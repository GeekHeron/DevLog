1. 数据采集层 (Ingestion Layer)

利用 Rust 编写的高性能 Scraper 定时抓取链上 RWA 协议（如 Centrifuge, Ondo）的实时数据。通过定制化的 Oracle 机制平衡链下宏观数据与链上 TVL 波动。

// 伪代码：RWA 协议监听逻辑 fn monitor_yield(protocol_addr: Address) { let logs = provider.get_logs(Filter::new().address(protocol_addr)).await?; let yield_rate = parse_apy(logs); push_to_vector_db(yield_rate); }

2. 双 Agent 决策引擎 (The Dual-Agent Core)

这是 GlobalYield AI 的核心壁垒：

  • Monitor Agent (Python/Pandas): 负责高频数据的利差计算，当跨协议溢价超过阈值时触发信号。
  • Analyst Agent (GPT-4o/Claude): 负责非结构化法律文档的语义分析，识别资产底层的潜在法律风险。

3. 风控模型 (GY-Score Model)

基于 Milvus 向量数据库存储的千万级法律文档向量，结合实时违约数据，生成动态的安全评估评分（0-100）。
